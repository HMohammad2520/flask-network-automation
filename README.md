## 📢 مقدمه

این پروژه یک پلتفرم سبک و انعطاف‌پذیر برای **اتوماسیون فرآیندهای شبکه و اجرای Script‌های سفارشی** است. ایده‌ی اصلی سیستم این است که به‌جای نوشتن ابزارهای پیچیده یا وابسته به زیرساخت، کاربر بتواند به‌سادگی منطق موردنظر خود را در قالب یک Script تعریف کرده و آن را در سناریوهای مختلف اجرا کند.

در این سیستم، Script‌ها از داده‌های اجرایی جدا شده‌اند و از طریق مفهومی به نام **Context**، در شرایط و محیط‌های مختلف قابل استفاده مجدد هستند. این طراحی باعث می‌شود بتوان یک Script واحد را در چندین سناریوی متفاوت بدون تغییر در کد استفاده کرد.

---

## ☰ کاربردها و سناریوهای استفاده

این سیستم برای طیف وسیعی از کاربردهای عملی در حوزه شبکه و DevOps مناسب است، از جمله:

### 1. اتوماسیون وظایف تکراری

کارهایی مانند:

* اجرای دستورات روی چندین سرور
* جمع‌آوری اطلاعات از سیستم‌ها
* بررسی وضعیت سرویس‌ها

می‌توانند به‌صورت یک Script تعریف شده و بارها اجرا شوند، بدون نیاز به اجرای دستی.

---

### 2. مدیریت چند محیط (Multi-Context Execution)

با استفاده از Context، می‌توان:

* یک Script را روی چند سرور با IPهای مختلف اجرا کرد
* پارامترهای مختلف (مثل credential، endpoint و...) را تغییر داد
* بدون تغییر کد، رفتار اجرا را کنترل کرد

این قابلیت به‌خصوص در محیط‌های staging، production و lab بسیار کاربردی است.

---

### 3. ساخت Workflowهای ساده

با ترکیب Script‌ها و Contextها می‌توان:

* فرآیندهای چندمرحله‌ای را خودکار کرد
* عملیات وابسته به هم را در قالب اجرای پشت‌سرهم Script‌ها پیاده‌سازی کرد
* به‌نوعی یک Automation pipeline سبک ساخت

---

### 4. تست و اجرای سریع ایده‌ها

به‌جای توسعه یک سرویس کامل:

* می‌توان یک Script نوشت
* آن را در سیستم ثبت کرد
* و بلافاصله اجرا و نتیجه را بررسی کرد

این موضوع برای prototyping و تست بسیار مفید است.

## طراحی یک نمونه 

در این مثال، یک Script ساده برای بررسی دسترسی (ping) به چند سرور در شبکه نوشته شده است. این Script از Context برای دریافت لیست IPها استفاده می‌کند.

### 🎯 هدف:

بررسی آنلاین بودن چند سرور و برگرداندن نتیجه

---

### 🌍 Code:

```python
import subprocess

class NetworkCheckScript:
    def run(self, **context):
        hosts = context.get("hosts", '')
        if not hosts:
            return f"Hosts missing from context: {hosts=}"

        results = {}

        for host in hosts.split(','):
            try:
                result = subprocess.run(
                    ["ping", "-c", "1", host.strip()],
                    stdout=subprocess.PIPE,
                    stderr=subprocess.PIPE,
                    timeout=3
                )
                results[host] = result.returncode == 0

            except Exception as e:
                results[host] = str(e)

        return {
            "status": "completed",
            "results": results
        }


instance = NetworkCheckScript()
```

---

### 📤 Context

```json
{
  "hosts": "8.8.8.8, 1.1.1.1, 192.168.1.1"
}
```

---

### 📤 Output

```json
{
  "status": "completed",
  "results": {
    "8.8.8.8": true,
    "1.1.1.1": true,
    "192.168.1.1": false
  }
}
```
