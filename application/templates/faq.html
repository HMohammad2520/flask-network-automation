🚀 Getting Started

Getting started with Devixa tools is simple! Here's a quick guide:

  1. Browse our repositories on GitHub to find the tool you need
  2. Install via pip (for Python packages):
pip install classmods

Or for py-zammad-cti:

pip install py-zammad-cti
  1. Check the documentation in each repository's README
  2. Explore examples to see the tool in action

All our tools are designed to be lightweight and easy to integrate into existing projects.

We keep dependencies minimal by design. Most Devixa tools have zero or very few external dependencies. Requirements are always listed in the requirements.txt or pyproject.toml file of each repository.

For example:

  • classmods - No external dependencies
  • py-zammad-cti - Only requires requests library

🔓 Open Source

Yes, absolutely! All Devixa Tech projects are 100% open source.

Our repositories are publicly available on GitHub, and you can:

  • View all source code
  • Fork and modify for your needs
  • Use in commercial projects (MIT License)
  • Learn from our implementation patterns

We believe transparency and community collaboration make better software.

Most Devixa projects use the MIT License, which is one of the most permissive open source licenses.

This means you can:

  • ✅ Use the software commercially
  • ✅ Modify the source code
  • ✅ Distribute your modifications
  • ✅ Use in proprietary projects

The only requirement is to include the original copyright notice. Check each repository for its specific license terms.

🤝 Contributing

We welcome contributions of all kinds! Here's how you can help:

  • 🐛 Report bugs - Open an issue on GitHub
  • 💡 Suggest features - Start a discussion or open a feature request
  • 📝 Improve documentation - Fix typos, add examples, clarify explanations
  • 💻 Submit code - Fork a repository, make changes, and submit a Pull Request
  • 🌍 Spread the word - Star our repos, share with colleagues, write about us

Check our Contributing Guide for detailed instructions and coding standards.

To ensure quality and consistency, please follow these guidelines:

  1. Keep it focused - One feature or fix per PR
  2. Write tests - If applicable, include tests for your changes
  3. Update documentation - Keep README and docstrings current
  4. Follow style - Match existing code style (PEP 8 for Python)
  5. Describe your changes - Explain what and why in the PR description

All contributions will be reviewed by maintainers. Don't hesitate to ask questions!

⚙️ Technical

We support Python 3.8 and above for all our Python packages.

Testing is performed regularly on:

  • Python 3.8
  • Python 3.9
  • Python 3.10
  • Python 3.11
  • Python 3.12

We recommend using the latest stable Python version for best performance and security.

For security concerns, please follow responsible disclosure:

  1. DO NOT open a public GitHub issue
  2. Email us directly at security@devixa.tech
  3. Provide detailed information about the vulnerability
  4. Allow us time to investigate and patch (usually 7-14 days)

We take security seriously and will acknowledge your report within 48 hours.

🎧 Support

  • Documentation - Check README files in each repository

We aim to respond to all inquiries within 24-48 hours (Monday-Friday).
